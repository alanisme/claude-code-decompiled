# Claude Code 的远程控制体系：本地工具栈的远程开关

逆向分析 Claude Code 的过程中，发现了一套相当完善的远程控制基础设施。Anthropic（以及企业管理员）可以在**不需要用户明确同意**的情况下远程修改工具行为。这套机制的覆盖面之广超出了预期。

## 远程托管设置：每小时一次的静默检查

### 工作原理

每个符合条件的 Claude Code 会话会定期从服务端拉取配置：

```
GET /api/claude_code/settings
```

源码位置: `src/services/remoteManagedSettings/index.ts`

### 轮询频率

```typescript
const POLLING_INTERVAL_MS = 60 * 60 * 1000 // 每小时一次
const DEFAULT_MAX_RETRIES = 5
```

每小时静默轮询一次，失败最多重试 5 次。这意味着 Claude Code 在后台每小时都会跟 Anthropic 的服务器通信一次。

### 谁会被纳入

- Console 用户（API key）: 全部纳入
- OAuth 用户: 仅 Enterprise/C4E 和 Team 订阅

### "要么接受，要么滚蛋"

这是最意外的设计。当远程推送的设置包含「危险」变更时，会弹出一个阻塞式对话框——但这个对话框只提供两个选择：

```typescript
// src/services/remoteManagedSettings/securityCheck.tsx:67-73
export function handleSecurityCheckResult(result: SecurityCheckResult): boolean {
  if (result === 'rejected') {
    gracefulShutdownSync(1)  // 退出码 1，直接终止
    return false
  }
  return true
}
```

拒绝远程设置？**程序直接退出。** 没有「暂时跳过」，没有「下次再说」。接受，或者 Claude Code 关掉。

企业级产品需要策略执行能力可以理解，但这个设计确实比较强硬。

### 缓存策略：一旦上钩就下不来

还有一个细节：远程服务器不可达时，系统会使用之前缓存的设置继续运行。也就是说，一旦 Claude Code 接受过远程设置，就算后来断网了，那些设置依然生效。**永远无法完全摆脱远程控制的影响。**

## 模型覆盖：远程换模型

Anthropic 还可以远程覆盖内部员工使用的模型：

```typescript
// src/utils/model/antModels.ts:32-33
// @[MODEL LAUNCH]: Update tengu_ant_model_override with new ant-only models
```

`tengu_ant_model_override` 这个 GrowthBook flag 能做的事挺多的：
- 设置默认模型
- 设置默认思考力度（effort level）
- 追加系统提示词
- 定义自定义模型别名

虽然目前只针对内部员工，但这套基础设施要扩展到外部用户技术上没有任何障碍。
